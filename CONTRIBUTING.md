# Contributing to AEGIS

Thank you for your interest in contributing to AEGIS.

Contributions that improve functionality, performance, documentation, or code quality are always welcome.

## How to Contribute

1. Fork the repository.
2. Create a new feature or fix branch.
3. Make your changes.
4. Test your changes before submitting.
5. Commit your changes with a clear commit message.
6. Push your branch to your fork.
7. Open a Pull Request describing your changes.

## Coding Guidelines

* Follow Python best practices (PEP 8).
* Write clean, modular, and well-documented code.
* Keep functions focused on a single responsibility.
* Avoid committing unnecessary files such as virtual environments, databases, logs, or API keys.

## Reporting Issues

If you discover a bug or have a feature request, please open an issue and include:

* A clear description of the problem.
* Steps to reproduce (if applicable).
* Expected behavior.
* Screenshots or logs (if helpful).

Thank you for helping improve AEGIS.
